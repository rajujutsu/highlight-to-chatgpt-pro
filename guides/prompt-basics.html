Prompt Basics: The 4-part formula

If your output is inconsistent, it’s usually because the prompt is missing one of these: Role, Task, Constraints, or Output Format.

Copy-ready

These templates are designed for Courier Web highlights: paste as a button prompt and it works immediately.

The formula

Role — who the model is

Task — what to do with the highlight

Constraints — tone, length, audience, “do-not” rules

Output format — bullets, table, JSON, steps, etc.

Why it works

You remove ambiguity. The model stops guessing the style and structure — and starts following a repeatable pattern.

Template: universal prompt (paste this)

ROLE: You are a helpful assistant. TASK: Use the highlighted text below to complete the task: {TASK_NAME} CONSTRAINTS: - Keep it concise and correct. - If the highlight is missing context, ask 1 clarification question at the end. - Do NOT invent facts. OUTPUT FORMAT: - {FORMAT} (use headings and bullets when helpful) HIGHLIGHT: {HIGHLIGHT}

Example: Summarize (button prompt)

ROLE: You are an expert summarizer. TASK: Summarize the highlighted text. CONSTRAINTS: - 5 bullets max - Plain language - Include 1 key takeaway OUTPUT FORMAT: - Bullets - Final line: "Key takeaway: ..." HIGHLIGHT: {HIGHLIGHT}

Example: Extract action items (button prompt)

ROLE: You are an operations assistant. TASK: Extract action items from the highlighted text. CONSTRAINTS: - If no actions exist, say "No clear action items." - Keep each action under 12 words. OUTPUT FORMAT: - Action items (checkbox list) - Owners (if mentioned) - Deadlines (if mentioned) HIGHLIGHT: {HIGHLIGHT}

Upgrade: add “style controls”

Add one line like: “Write in a direct, confident tone. No fluff.” That alone makes outputs feel more human and consistent.
