If you must reference it, redact it: `ABCD-****-****-WXYZ` --- ## 💻 Device limit (2 devices per license) Each license can be used on up to **2 devices**. You may hit the limit if you: - switched computers - reinstalled your OS - created a new Chrome profile - cleared browser storage - reinstalled the extension If you see a **device_limit** error, use **Reset devices** below. --- ## 🔄 Reset devices (30-day cooldown) If you got a new device (or lost access to an old one), you can reset device slots. ✅ **Reset clears the device list** for your license ✅ After reset, you can activate on **up to 2 devices** again ⏳ Reset is limited to **once every 30 days** (cooldown) **Where to reset** - Open the extension **Options** - Go to **Pro / License** - Click **Reset devices** If you see a cooldown message, you’ll be shown the next eligible reset time. --- ## ✅ Activation checklist If Pro won’t activate: - Paste the key exactly (no spaces before/after) - Confirm you’re online - Try again in 60 seconds (rate limit protection) - If you see **device_limit**, do **Reset devices** - If you see **invalid_token**, follow the steps in Troubleshooting --- ## 🧰 Troubleshooting ### I get `409 (Conflict)` from `/verify-license` That usually means **device_limit** (you’re trying to activate a 3rd device). **Fix:** 1. Use **Reset devices** (Options → Pro / License) 2. Activate again with your key --- ### I get `401 Invalid JWT` / `{ ok:false, reason:"invalid_token" }` This means the request is sending a token that your server can’t validate. **Most common causes:** - `H2C_JWT_SECRET` changed (tokens minted with the old secret won’t validate) - Token is missing / not being passed as `Authorization: Bearer ` - Token expired (check `h2c_pro_token_exp`) **Fix (fast):** 1. In extension Options → **Log out / Reset Pro** 2. Re-activate with your license key (this mints a fresh token) --- ### Floating buttons / shortcut not working - Make sure the extension is enabled - Reload the page - Confirm you’re selecting enough text (minimum selection setting) - Some pages are blocked (`chrome://`, extension pages, etc.) --- ## 📨 Contact support / Open an issue Best support channel: **GitHub Issues**. ➡️ Open a new issue: - https://github.com/YOUR_USERNAME/YOUR_REPO/issues/new **Please include:** - What you expected to happen - What actually happened - Browser + OS (example: Chrome 122 / macOS 14) - The exact error message - Screenshot (optional) - Redacted license key (optional): `ABCD-****-****-WXYZ` --- ## 🧾 Issue template (copy/paste) ```text **What happened?** … **Steps to reproduce** 1) 2) 3) **Browser / OS** Chrome version: OS: **Error message** (paste here) **License key (redacted)** ABCD-****-****-WXYZ **Anything else** …
